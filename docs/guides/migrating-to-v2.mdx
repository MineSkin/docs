The MineSkin API V2 introduces some changes to how skins are generated. This guide will outline the most important changes to help you migrate to the new endpoints.

## Changes

### Queue System

The new API uses a queue system to handle skin generation requests.
This means instead of making a request and receiving back the skin data after a few seconds, you will now receive the ID of a [queued job](../mineskin-api/queue-skin-generation) which you can periodically [check the status](../mineskin-api/get-job-status) of.
